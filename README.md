# TektonBilling Organization

Welcome to the TektonBilling organization! This `.github` repository contains default community health files and templates that are automatically applied across all repositories in the TektonBilling organization.

## üìã What is This Repository?

This is a special `.github` repository that GitHub recognizes at the organization level. Any files placed here will serve as defaults for all repositories in the organization that don't have their own versions of these files.

## üìÅ Repository Contents

### Pull Request Template

**Location**: `.github/pull_request_template.md`

This template is automatically applied when creating pull requests across any repository in the TektonBilling organization. It includes the following sections:

- **Summary**: Brief description of the changes
- **Context**: Explanation of why the change is required and related issue references
- **Changes**: List of major modifications made in the PR
- **Database Changes**: Documentation of any schema changes
- **Testing**: How the changes were tested, including manual test steps and screenshots
- **Additional Notes**: Any extra information relevant to the PR

## üéØ Purpose

This repository helps maintain consistency across the TektonBilling organization by:

1. **Standardizing Pull Requests**: Ensuring all PRs follow a consistent format with necessary information
2. **Improving Communication**: Making it easier for team members to understand changes
3. **Streamlining Reviews**: Providing reviewers with all the context they need
4. **Maintaining Quality**: Encouraging thorough documentation and testing

## üöÄ Usage

### For Contributors

When you create a pull request in any TektonBilling repository, the PR template will automatically populate. Simply fill out each section with relevant information about your changes.

### For Maintainers

To add or modify organization-wide templates:

1. Clone this repository
2. Add or modify files in the `.github` directory
3. Create a pull request with your changes
4. Once merged, the changes will automatically apply to all repositories in the organization
